\section{Preliminaries}

\subsection{Temporal queries}

In this paper we adopt the
\emph{discrete linear time} model~\cite{BohlenDGJ_ebiss2017_temporal-data-management},
where the time domain is an ordered set of discrete \emph{chronons}.
For the continuous case we can simply choose as a chronon an interval of length
equal to the greatest common divisor of the intervals in all relations, and use
it to quantize all the intervals.
A time interval $e = [)$ is a contiguous set of chronons between a starting
time $e_s$ (included) and and ending time $e_e$ (excluded), that is the intervals are \emph{close open}.
Given an interval $e$, we denote with $|e| = e_e - e_s$ its duration, i.e. the number of chronons it spans.
Furthermore, given two intervals $e$ and $e'$ we denote with $e \cap e'$ the set of chronons
shared by the two intervals, which might be empty.
Note that such an intersection is itself an interval, if not empty.

Throughout this paper we will refer to sets of intervals rather than relations with time interval attributes,
for simplicity of exposition.

Our index efficiently supports the three following types of queries (defined as in~\cite{BehrendDGSVRK_SSTD19_period-index}).

\cetodo{Shall we rename it \emph{temporal query?}. Ultimately, also duration queries involve a range of durations, so it might be confusing.}
\begin{definition}[Range query]
    Given a temporal interval $e = [e_s, e_e)$ and a set of intervals $A$,
    a \emph{range query} $Q(A, e)$ is defined as
    \[
        Q(A, e) = \{e' | e' \in A \wedge e' \cap e \ne \emptyset \}
    \]
\end{definition}

\begin{definition}[Duration query]
    Given a duration interval $d = [d_{min}, d_{max}]$ and a set of intervals $A$,
    a \emph{duration query} $Q(A, d)$ is defined as
    \[
        Q(A, d) = \{e | e \in A \wedge |e| \in [d_{min}, d_{max}] \}
    \]
\end{definition}

\begin{definition}[Range-duration query]
    Given a temporal interval $e = [e_s, e_e)$, a duration interval $d = [d_{min}, d_{max}]$,
    and a set of intervals $A$,
    a \emph{range-duration query} $Q(A, e, d)$ is defined as
    \[
        Q(A, e, d) = \{e' | e' \in A \wedge e' \cap e \ne \emptyset \wedge |e'| \in [d_{min}, d_{max}] \}
    \]
\end{definition}

The latter type of query is a combination of the former two.

\section{Data structure}

Our index is a grid, partitioning the intervals into disjoint buckets according
to the durations and the start times of the intervals.
The boundaries between cells are defined by using the empirical cumulative
distribution function of the duration and the starting times, so that
each cell contains the same number of intervals, which corresponds to the
\emph{page size} of our index.

Therefore the structure is adaptive to the distribution of the input.

\subsection{Index construction}

Let $s$ be the \emph{page size} parameter.
For an interval $e \in A$, let $CDF_d^A(|e|)$ be the fraction of intervals $e' \in A$
whose duration is
$\le |e|$.
Similarly, let $CDF_s^A(e_s)$ be the fraction of the
intervals whose start time is $\le e_s$.

\cetodo{What happens when there are more than $s^2$ elements in a bucket?
    This happens when there are more than $s^2$ elements with the same duration: they
    all fall in the same bucket.}
\cetodo{We need to describe a way to efficiently compute the cumulative distribution function.}
We partition the intervals in $n / s^2$ buckets: an interval $e$ is assigned to
bucket $\lfloor CDF_d(|e|) \cdot n/2^s \rfloor$.
In particular, we define the $i$-th bucket in the duration dimension as
\[
    A_i^d = \{ e \in A | i = \lfloor CDF_d^A(|e|) \cdot |A|/2^s \rfloor \}
\]
Consider one such bucket $i$, and let $[\bar{d}_{min}, \bar{d}_{max}]$ be the
range of durations spanned by the intervals it contains.
We further partition the intervals in $A_i^d$ according to the distribution
of their starting times:
\[
    A_{i,j}^{d,s} = \{ e \in A_i^d ~|~ j = \lfloor CDF_s(e_s) \cdot |A_i^d|/s \rfloor \}
\]

Our index is a grid built out of the cells $A_{i,j}^{d,s}$.
In each cell, the intervals are sorted by decreasing end time.

We can use a compact representation of the cumulative density function
that is sufficient for indexing into this grid.
Observe that since we always apply a floor operator to the result of the CDF
we can just store the values of the CDF defining the start of a new bucket.

\subsection{Querying the index}

Given a range-duration query $Q(A, e, d)$ with $e=[e_s, e_e)$ and $d=[d_min, d_max]$,
recall that an interval $e'$ satisfies the query if the three following conditions
are met:
\begin{align}
     & |e'| \in [d_{min}, d_{max}] \label{eq:duration-condition} \\
     & e_s < e'_e                  \label{eq:start-condition}    \\
     & e'_s < e_e                  \label{eq:end-condition}
\end{align}

First, find the two indices which identify the range of cells in the duration
dimension each containing at least one interval satisfying
condition~\eqref{eq:duration-condition}.
\begin{align*}
    i_{start} & = \left\lfloor CDF_d^A(d_{min}) \cdot |A| / s^2 \right\rfloor \\
    i_{end}   & = \left\lfloor CDF_d^A(d_{max}) \cdot |A| / s^2 \right\rfloor
\end{align*}
Then, for each $i \in [i_{start}, i_{end}]$, let $[d_{min}^i, d_{max}^i]$
be the range of durations spanned by the $i$-th column.
We consider the cells in the start
dimension in the range $[j_{start}, j_{end}]$ with
\begin{align*}
    j_{start} & = \left\lfloor CDF_{s}^{A_i^d}(e_s - d_{max}^i) \cdot |A_i^d| / s \right\rfloor \\
    j_{end}   & = \left\lfloor CDF_{s}^{A_i^d}(e_e) \cdot |A_i^d| / s \right\rfloor
\end{align*}
In other words,
the start is the first cell such that the upper boundary on the start
time plus the maximum duration in the duration range of the column
is strictly less than the end time of the query
(in Figure~\ref{fig:grid-query} this corresponds to the upper corner of each cell).

For each cell $A_{i,j}^{d,s}$ within the above bounds, we evaluate the query
predicate for all the intervals. Since the intervals are sorted by decreasing
end time, we can stop as soon as we see an interval whose en time is $< e_s$.

Figure~\ref{fig:grid-query} shows an example of a grid together with a query.
Note that each group of durations is partitioned according to start times
in a different way.
As for the query, the dashed lines mark the range of intervals overlapping
the query intervals. Intersecting this area with the diagonals enclosing the
duration part of the query gives the area where query answers lie.

\subsection{Analysis}

\begin{theorem}
    Given a set of $n$ intervals, building the index requires time $\BO{n \log n}$.
\end{theorem}
\begin{proof}
    Computing the Empirical Cumulative Distribution Function for the duration measure can be done
    by first sorting the intervals according to the duration and then scanning through the sorted interval
    to compute the cumulative counts. A final scan allows to associate each interval to appropriate bucket.
    These operations can thus be accomplished in time $\BO{n \log n}$.
    Each bucket is then further partitioned according to the start times.
    Let $n_i$ be the size of the $i$-th bucket.
    Then, partitioning it according to the cumulative distribution function
    requires time $\BO{n_i \log n_i}$, following the same argument as above.
    Therefore, the time for building the second level of the index is
    \[
        \BO{\sum_i n_i \log n_i}
        = \BO{\log n \cdot \sum_i n_i}
        = \BO{n \log n}
    \]
\end{proof}

The following theorem ensures that, by setting the page size $s$ appropriately, the query time
can be made polynomially smaller than the input size.
\begin{theorem}
    Given an index over a set of $n$ intervals, with page size $s$, the time for
    answering a range-duration query is
    \[
        \BO{\frac{n}{s} + s^2 + k}
    \]
    where $k$ is the number of intervals matching the query predicate.
    For $s = \BO{n^\epsilon}$, we have that the query time is
    \[
        \BO{n^\epsilon + k}
    \]
\end{theorem}
\begin{proof}
    Main points of the proof
    \begin{itemize}
        \item Define \emph{heavy} and \emph{light} columns, with the former being columns with more than $s^2$ intervals.
              All intervals in a heavy column share the same duration.
        \item Within columns, define \emph{heavy} and \emph{light} cells, with the former having more than $s$ intervals in them.
              All intervals in a heavy cell share the same starting point. Question: can we even have heavy cells?
              I mean, we are under set semantics, so there are no duplicate intervals. In a heavy column, necessarily all
              the intervals \emph{must} have distinct starting times, lest there are duplicate intervals. So there \emph{cannot} be
              heavy cells in heavy columns.
              What about heavy cells in light columns? That would mean that there are $> s$ intervals with the same starting points
              but with distinct durations.
              Since the bucket size for distances is $s^2$, all these intervals can be indeed fit into the same bucket
              (think of intervals starting at 0 with durations $1, 2, \dots, s^2$).
              So the answer is yes, you can have them but only in light columns.
    \end{itemize}
\end{proof}
